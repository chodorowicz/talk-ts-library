## Rollup and microbundle enforced config

```
noEmitHelpers: false
importHelpers: true // required tslib, provided by rollup-plugin-typescript2

noEmit: false
inlineSourceMap: false
outDir: taken from package.json `main`
target: 'esnext' // because it's transpiled by babel
declaration: true // generate d.ts files
sourceMap: true
```

## What does it do?

- Ensures you are publishing from the master branch
- Ensures the working directory is clean and that there are no unpulled changes
- Reinstalls dependencies to ensure your project works with the latest dependency tree
- Runs the tests
- Bumps the version in package.json and npm-shrinkwrap.json (if present) and creates a git tag
- Publishes the new version to npm
- Pushes commits and tags to GitHub/GitLab
- Opens a prefilled GitHub Releases draft after publish

## Backward types compability

- use `typesVersions` (available since TS 3.1)

```ts
{
  "name": "package-name",
  "version": "1.0",
  "types": "./index.d.ts",
  "typesVersions": {
    ">=3.2": { "*": ["ts3.2/*"] },
    ">=3.1": { "*": ["ts3.1/*"] }
  }
}
```
